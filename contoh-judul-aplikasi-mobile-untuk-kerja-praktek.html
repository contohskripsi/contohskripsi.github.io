12. bagaimana teknik pengujiannya aplikasi mobile


Jawaban:

Testing aplikasi mobile adalah proses dimana aplikasi yang dikembangkan untuk perangkat mobile diuji untuk kegunaan dan konsistensinya. Ada dua macam pengujian yang harus dilakukan untuk aplikasi pada perangkat mobile yaitu pengujian hardware dan software, dan berikut adalah pembahasanya.

Testing hardware

Hardware termasuk prosesor, memori internal, ukuran layar, resolusinya, besarnya RAM, kemampuan kamera, bluethooth, WIFI dan lain-lain akan menjadi komponen yang yang harus diuji dalam proses pengujian ini.

Testing Software

Dalam tahap ini kemudian aplikasi mobile yang telah dikembangkan akan diuji secara mendetail dan teliti. Fungsi dan konsistenti sebuah aplikasi mbile akan diuji. Seperti kita ketahui bahwa aplikasi mobile terbagi menjadi tiga macam yakni: aplikasi mobile native, aplikasi mobile hybrid dan aplikasi mobile web. Ketiga nya memiliki perbedaan dasar yang tentunya akan mempengaruhi proses pengujian

17. 1. Jelaskan perbedaan aplikasi berbasis web dan aplikasi berbasis mobile serta jelaskan bagaimana sebuah perusahaan menentukan kapan harus menggunakan aplikasi berbasis web dan kapan harus menggunakan aplikasi berbasis mobile!


Aplikasi berbasis web dan aplikasi berbasis mobile memiliki perbedaan dalam hal platform, aksesibilitas, pengembangan, dan pengalaman pengguna. Berikut adalah penjelasan tentang perbedaan antara keduanya:

1. Platform:

- Aplikasi berbasis web dirancang untuk diakses melalui browser web dan dapat dijalankan di berbagai platform, seperti desktop, laptop, tablet, dan perangkat seluler.

- Aplikasi berbasis mobile dirancang khusus untuk dijalankan pada perangkat seluler seperti smartphone atau tablet.

2. Aksesibilitas:

- Aplikasi berbasis web dapat diakses dengan mudah melalui URL dan tidak memerlukan instalasi tambahan pada perangkat pengguna. Pengguna hanya perlu memiliki browser web yang kompatibel.

- Aplikasi berbasis mobile harus diunduh dan diinstal secara khusus dari toko aplikasi (misalnya, App Store untuk iOS atau Google Play Store untuk Android) sebelum pengguna dapat menggunakannya.

3. Pengembangan:

- Aplikasi berbasis web biasanya dikembangkan menggunakan teknologi web seperti HTML, CSS, dan JavaScript. Pengembang dapat menggunakan kerangka kerja (framework) seperti React, Angular, atau Vue.js untuk mempermudah pengembangan.

- Aplikasi berbasis mobile memerlukan pengembangan menggunakan bahasa pemrograman yang spesifik untuk platform tertentu, seperti Swift atau Objective-C untuk iOS, dan Java atau Kotlin untuk Android.

4. Pengalaman Pengguna:

- Aplikasi berbasis web memiliki desain responsif yang dapat menyesuaikan tampilan dengan baik pada berbagai perangkat. Namun, pengalaman pengguna mungkin berbeda-beda tergantung pada browser yang digunakan dan kualitas koneksi internet.

- Aplikasi berbasis mobile dapat memberikan pengalaman pengguna yang lebih terintegrasi dengan perangkat seluler, termasuk akses ke fitur-fitur perangkat seperti kamera, GPS, notifikasi push, dan lainnya.

Keputusan perusahaan untuk menggunakan aplikasi berbasis web atau aplikasi berbasis mobile tergantung pada beberapa faktor, seperti:

1. Tujuan Aplikasi:

- Jika aplikasi bertujuan untuk menyediakan informasi umum yang mudah diakses, seperti situs web perusahaan, blog, atau platform e-commerce, aplikasi berbasis web mungkin lebih cocok.

- Jika aplikasi dirancang untuk memberikan pengalaman yang kaya dan terintegrasi dengan fitur-fitur perangkat seluler, seperti aplikasi perbankan, jejaring sosial, atau permainan mobile, maka aplikasi berbasis mobile lebih disarankan.

2. Target Pengguna:

- Jika target pengguna aplikasi adalah pengguna yang menggunakan berbagai platform dan perangkat, aplikasi berbasis web dapat mencakup lebih banyak pengguna potensial, karena dapat diakses dari berbagai perangkat.

- Jika target pengguna aplikasi adalah pengguna yang menggunakan perangkat seluler secara eksklusif, seperti aplikasi berbasis lokasi atau aplikasi berbasis kamera, maka aplikasi berbasis mobile dapat memberikan pengalaman yang lebih terarah dan terintegrasi.

3. Fitur dan Kebutuhan Teknis:

- Jika aplikasi memerl

ukan akses ke fitur-fitur perangkat seluler khusus, seperti GPS, kamera, sensor gerak, atau pemrosesan offline, maka aplikasi berbasis mobile menjadi pilihan yang lebih baik.

- Jika aplikasi hanya memerlukan tampilan dan interaksi dasar tanpa memerlukan akses ke fitur perangkat seluler, maka aplikasi berbasis web dapat memenuhi kebutuhan tersebut dengan lebih efisien.

Perusahaan perlu menganalisis kebutuhan, tujuan, dan karakteristik pengguna aplikasi mereka untuk menentukan apakah aplikasi berbasis web atau aplikasi berbasis mobile lebih cocok dalam konteks yang diberikan. Dalam beberapa kasus, perusahaan juga dapat mempertimbangkan untuk mengembangkan aplikasi hibrida yang menggabungkan kelebihan dari kedua platform tersebut.

Jawaban:

Aplikasi berbasis web dan aplikasi berbasis mobile memiliki perbedaan dalam platform yang digunakan dan cara aksesnya:

Aplikasi Berbasis Web:

Aplikasi berbasis web dirancang untuk diakses melalui browser web di perangkat yang terhubung ke internet.

Aplikasi ini dapat diakses melalui berbagai perangkat, seperti komputer, laptop, tablet, dan smartphone.

Pengguna tidak perlu menginstal aplikasi secara khusus, cukup membuka browser dan mengakses situs web yang menyediakan aplikasi tersebut.

Pengembangan dan pemeliharaan aplikasi web relatif lebih sederhana karena hanya perlu fokus pada satu platform, yaitu web.

Aplikasi Berbasis Mobile:

Aplikasi berbasis mobile dirancang khusus untuk dijalankan di perangkat mobile, seperti smartphone dan tablet.

Aplikasi ini harus diunduh dan diinstal secara khusus di perangkat pengguna melalui toko aplikasi (App Store untuk iOS, Google Play Store untuk Android).

Aplikasi mobile biasanya menawarkan pengalaman yang lebih dioptimalkan untuk perangkat mobile dengan fitur dan antarmuka yang dirancang khusus untuk layar sentuh dan mobilitas pengguna.

Keputusan untuk menggunakan aplikasi berbasis web atau aplikasi berbasis mobile tergantung pada beberapa faktor, seperti:

Tujuan Aplikasi: Jika aplikasi bertujuan menyediakan informasi dan layanan yang dapat diakses oleh banyak pengguna melalui berbagai platform dan perangkat, aplikasi berbasis web lebih cocok.

Ketergantungan pada Fitur Perangkat: Jika aplikasi membutuhkan akses ke fitur perangkat mobile tertentu, seperti GPS, kamera, atau sensor lainnya, maka aplikasi berbasis mobile akan lebih tepat.

Pengalaman Pengguna: Jika aplikasi perlu memberikan pengalaman yang dioptimalkan dan kustomisasi sesuai dengan perangkat mobile, seperti antarmuka sentuh dan notifikasi push, aplikasi berbasis mobile dapat memberikan pengalaman yang lebih baik.

Ketersediaan dan Aksesibilitas: Jika aplikasi harus dapat diakses dengan mudah tanpa memerlukan instalasi khusus, aplikasi berbasis web lebih praktis karena dapat diakses melalui browser.

Jawaban:

Cara Membuat Aplikasi Mobile

Berikut ini adalah beberapa panduan untuk memulai aplikasi seluler yang perlu Anda lakukan.

Langkah 1: Dapatkan ide atau masalah yang dihadapi saat ini.

Hal ini akan membuat Anda berpikir kreatif untuk mendapatkan sebuah ide dalam membuat aplikasi mobile yang Anda inginkan. Lihatlah sekeliling Anda, sebab dalam setiap produk dan layanan yang Anda gunakan saat ini semuanya diciptakan untuk memecahkan semua masalah Anda.

Karena itu carilah masalah dalam kehidupan sehari-hari lalu masukkan semua masalah tersebut ke dalam suatu daftar masalah. Jika telah lengkap, maka mulailah memikirkan bagaimana Anda dapat mengatasi masalah tersebut dengan cepat, efisien serta masuk akal.

Langkah 2: Identifikasi Kebutuhan.

Lakukan analisa untuk mengidentifikasi kebutuhan para pelanggan, serta seberapa banyak permintaan yang akan Anda dapatkan saat Anda membuat aplikasi tersebut. Anda dapat mencari ide Anda dengan menggunakan mesin pencari Google untuk dapat mengetahui seberapa banyak orang yang mencari tahu aplikasi apa yang mereka harapkan untuk membantunya. Anda juga dapat membuat sebuah forum yang bertemakan ide tentang aplikasi yang akan Anda buat serta mencari minat para pengguna melalui pendaftaran email.

Langkah 3: Meletakkan arus dan fitur

Validasikan ide aplikasi Anda yang menandakan bahwa Anda mempunyai sesuatu yang ingin orang lain gunakan. Sekarang merupakan waktu yang tepat untuk merinci produk Anda ke dalam sebuah dokumen, atau jika Anda ingin bekerja ekstra, Anda dapat menggunakan alat wireframing untuk mendukung pekerjaan Anda.

Ketika menggambarkan ide Anda pada sebuah kertas, jangan pernah lupa untuk melakukannya sedetail mungkin. Selain itu, sertakan juga cara pengguna untuk melakukan navigasi aplikasi serta semua fitur yang telah Anda bayangkan. Hal ini akan membantu Anda dalam mengembangkan serta memahami kemauan serta harapan Anda.

Langkah 4: Hapus semua fitur non-inti

Dari berbagai komponen serta fitur yang telah Anda siapkan, mulailah untuk melihat secara mendetail fitur apa yang dapat Anda hapus. Jangan pernah sekalipun membuat fitur bagus pada versi pertama, sebab fitur tersebut nantinya dapat Anda tambahkan sebagai pembaruan. Hal ini tentu untuk menjaga agar mengurangi biaya awal pembuatan aplikasi ini serta membantu aplikasi Anda untuk dapat lebih cepat ke pasar.

Langkah 5: Buat Desain Aplikasi

Buatlah desain aplikasi yang ingin Anda tampilkan pada aplikasi Anda, sebab desain bukan hanya mengenai bagaimana aplikasi Anda terlihat, namun juga mengenai perasaan para pengguna yang menggunakan aplikasi Anda. Karena itu, carilah pengembang aplikasi yang mampu mendesain aplikasi Anda dengan sangat baik.

aplikasi mobile

Langkah 6: Menyewa Desainer/ Pengembang

Untuk mengembangkan aplikasi seluler yang ingin Anda buat, maka carilah perusahaan pengembang yang mempunyai tim dengan desain yang hebat dan solid. Saat menyewa pengembang, carilah informasi sebanyak-banyaknya melalui internet untuk memeriksa kredibilitas pengembang tersebut serta aplikasi apa saja yang telah mereka buat. Jika dari portofolio yang telah mereka kerjakan Anda menyukainya, maka pengembang tersebut dapat Anda sewa untuk mengembangkan aplikasi Anda. Salah satu perusahaan pengembang aplikasi terpercaya adalah Logique Digital Indonesia.

Langkah 7: Melakukan Integrasi Analisis

Analitik berguna untuk membantu Anda dalam melacak berbagai unduhan, keterlibatan pengguna, dan retensi untuk aplikasi yang telah Anda buat. Sangat disarankan untuk menggunakan alat seprti Flurry dan Loyaltics yang mempunyai versi gratis dan berbayar.

Langkah 8: Dapatkan umpan balik dari pelanggan kemudian lakukan improvisasi.

Setelah aplikasi yang Anda buat telah berada di toko aplikasi, maka langkah pertama adalah mengamati perilaku pengguna dari aplikasi Anda. Sebab perilaku pelanggan akan memberikan informasi terbaik bagaimana cara Anda untuk terus memperbaiki dan meningkatkan aplikasi Anda.

Langkah 9: Perkenalkan Fitur.

Ketika aplikasi Anda telah diperkenalkan, maka jangan lupa untuk memberikan fitur terbatas pada versi pertama aplikasi Anda dengan hanya menawarkan inti dari aplikasi tersebut. Selanjutnya, terus lakukan evaluasi dan mengenalkan berbagai fitur yang tersisa dari versi awal untuk melakukan analisis dan mendapatkan umpan balik dari pelanggan mengenai fitur tersebut.
