16. Lapisan teknologi pada pengembangan perangkat lunak ini dapat dibagi menjadi 4 lapisan. Jelaskan yang dimaksud lapisan teknologi "Process Model" !


Jawaban:

Penjelasan:

Lapisan Perangkat Lunak atau Software Layer Secara Umum

Software layer atau lapisan perangkat lunak terdiri dari beberapa layer atau lapisan yakni a quality focus atau fokus kualitas, process atau proses, methods atau cara dan juga tools atau alat.

1. A Quality Focus [Fokus Kualitas]

Ketika anda sedang membangun aplikasi, maka fokus kualitas menjadi hal pertama yang harus anda bangun. Anda harus memikirkan tentang kualitas software yang mau anda bangun, siapa yang akan dijadikan sasaran, untuk siapa aplikasi tersebut dibangun. Untuk itulah, fokus kualitas akan membantu programmer untuk mengetahui level aplikasi yang akan dibangun.

Sebagai contoh jika anda ingin membuat aplikasi untuk memutar musik, maka dengan fokus kualitas programmer bisa mengetahui sudah sampai di mana aplikasi yang mau dibangun tersebut. Sedangkan untuk file music sendiri juga bisa bervariasi dari mulai MP2, MP3, WAV, audio track, MIDI dan lain sebagainya.

Dengan mengetahui tujuan dari dibuatnya aplikasi, maka programmer bisa mengerti tentang semua hal yang berkaitan dengan program tersebut. Sementara jika dilihat dari segi interaksi manusia serta komputer, maka fokus kualitas bisa membantu programmer tentang bentuk dari aplikasi yang ingin dibuat.

2. Process atau Proses

Proses atau process adalah lapisan nomor dua di dalam software layer yakni sesudah quality focus. Sesudah nantinya quality focus dibangun, maka pemrogram juga harus mengetahui cara memproses yang harus dilakukan berkaitan dengan fokus kualitas perangkat lunak yang diinginkan.

Beberapa proses ini dilakukan berurutan dan juga tepat untuk menghindari kesalahan pada saat aplikasi akan launching atau diluncurkan. Berbagai proses tersebut nantinya akan dilakukan sesuai dengan kata kuncinya.

3. Methods atau Metode

Methods atau metode adalah hal penting lainnya ketika membuat sebuah perangkat lunak atau software. Dengan memakai metode, maka programmer juga akan melakukan langkah serta tindakan sesuai dengan metode tersebut. Metode yang akan dipakai juga harus sesuai dengan perangkat lunak atau software yang ingin dibangun serta tujuan dari dibuatnya perangkat lunak tersebut.

4. Tools atau Alat

Tools adalah alat bantu yang bisa dipakai programmer ketika ingin menyelesaikan sebuah proyek. Dari mulai tools anmiasi, tool multimedia sampai tools normalisasi dan berbagai tools lain. Contohnya adalah power designer, X3D, paintshop pro dan sebagainya.

C. Fungsi Software Layer

Jika dilihat secara umum, fungsi dari software layer ini tidak jauh berbeda dengan beberapa bidang rekayasa lain dan berikut beberapa fungsi software layer selengkapnya:

Supaya biaya produksi software atau perangkat lunak bisa lebih rendah.

Bisa menghasilkan software perangkat lunak dengan kinerja yang tinggi, tepat waktu serta andal.

D. Jenis Arsitektur Perangkat Lunak

Jenis dari software layer terdiri dari data flow architectures, data centered architectures, call amd return architectures serta layered architectures, implicit invocation, event based, table driven interpreters, hetereogeneus archietecture.

Data Centered Architectures: Untuk mendapatkan kualitas integrability data yang baik sebagai tempat penyimpanan data yang terpusat.

Data Flow Architectures: mempunyai tujuan untuk mendapatkan kualitas pemakaian ulang serta modifiability yang ditandai dengan melihat sistem sebagai sebuah rangkaian transformasi di potongan berurutan input data. Data nantinya akan masuk ke dalam sistem dan mengalir dari satu komponen ke komponen lain dan datanya sendiri memiliki beberapa buah tujuan akhir.

Call and Return Architectures: Bertujuan untuk mendapatkan kualitas solvabilitas dan juga modifibiality.

Layer Architectures: Sistem yang berlapis dan diatur sedemikian rupa di mana masing masing lapisan menyediakan layanan ke lapisan lain diatasnya serta untuk melayani klien ke lapisan bawah.

Event Based, Implicit Invocation: Komponen antarmuka yang memberi koleksi prosedur serta fungsi.

Repositories: Pada respitori ada 2 macam komponen yakni pusat struktur data dan juga koleksi independen komponen untuk menyimpan data pusat. Sedangkan untuk interaksi antara repositori serta komponen eksternal bisa bervariasi diantara sistem.

23. apa fungsi SQL dalam pengembangan perangkat lunak​


SQL (Structured Query Language) adalah bahasa yang digunakan untuk mengelola dan mengakses basis data dalam pengembangan perangkat lunak. Fungsi SQL dalam pengembangan perangkat lunak sangat penting dan mencakup beberapa aspek utama, antara lain:

1. Manipulasi Data: SQL digunakan untuk mengambil, memasukkan, memperbarui, dan menghapus data dari basis data. Dengan SQL, pengembang dapat membuat, membaca, memperbarui, dan menghapus catatan data dalam aplikasi mereka.

2. Membangun dan Mengelola Basis Data: SQL memungkinkan pengembang untuk membuat basis data, tabel, indeks, dan relasi antara tabel. Ini adalah langkah penting dalam desain dan pengelolaan data dalam aplikasi.

3. Kueri Data: SQL memungkinkan pengembang untuk mengekstrak data yang diperlukan dari basis data dengan menggunakan pernyataan kueri (query). Ini penting untuk mengambil informasi yang dibutuhkan dari basis data yang besar.

4. Keamanan Data: SQL menyediakan mekanisme untuk mengontrol akses ke data. Ini termasuk pengaturan izin (permissions) untuk pengguna dan peran tertentu, sehingga hanya pengguna yang diizinkan yang dapat mengakses dan memanipulasi data.

5. Pengelolaan Transaksi: SQL mendukung transaksi, yang memungkinkan pengembang untuk mengelola operasi-operasi yang melibatkan perubahan data dalam basis data. Ini memastikan konsistensi data dan menghindari masalah seperti kehilangan data.

6. Penyimpanan Prosedur dan Fungsi: SQL memungkinkan pengembang untuk membuat penyimpanan prosedur (stored procedures) dan fungsi (functions) yang dapat digunakan dalam aplikasi. Hal ini membantu dalam pengelolaan logika bisnis di dalam basis data.

7. Integrasi dengan Aplikasi: SQL dapat diintegrasikan dengan banyak bahasa pemrograman dan kerangka kerja pengembangan perangkat lunak. Ini memungkinkan aplikasi untuk berinteraksi dengan basis data menggunakan SQL dalam kode mereka.

Dengan fungsi-fungsi ini, SQL menjadi komponen penting dalam pengembangan perangkat lunak yang berbasis data, memungkinkan pengembang untuk mengelola data dengan efisien dan aman.

31. Apa itu Flutter dan apa kegunaannya dalam pengembangan perangkat lunak?

Flutter adalah sebuah framework open-source yang dikembangkan oleh Google. Framework ini digunakan untuk membangun antarmuka pengguna (user interface) pada berbagai platform seperti Android, iOS, web, desktop, dan lain-lain.

Kegunaan Flutter dalam pengembangan perangkat lunak adalah sebagai berikut:

1. Pengembangan lintas platform: Dengan Flutter, Anda dapat mengembangkan aplikasi yang dapat berjalan di berbagai platform, termasuk Android, iOS, web, desktop, dan lain-lain. Hal ini mengurangi kebutuhan untuk mengembangkan aplikasi terpisah untuk setiap platform.

2. Antarmuka pengguna yang kaya: Flutter menyediakan berbagai komponen dan widget yang memungkinkan pengembang untuk membuat antarmuka pengguna yang menarik dan responsif. Widget tersebut dapat dengan mudah dikustomisasi dan dirancang sesuai dengan kebutuhan aplikasi.

3. Kinerja yang tinggi: Flutter menggunakan rendering langsung (direct rendering) yang memungkinkan aplikasi memiliki kinerja yang tinggi dan responsif. Hal ini karena Flutter dapat menggambar antarmuka pengguna langsung pada layar perangkat, tanpa menggunakan jembatan (bridge) seperti pada framework lainnya.

4. Hot Reload: Salah satu fitur unggulan Flutter adalah Hot Reload, yang memungkinkan pengembang untuk melihat perubahan yang dilakukan pada kode secara instan, tanpa perlu melakukan kompilasi ulang atau me-restart aplikasi. Fitur ini mempercepat proses pengembangan dan memungkinkan pengembang untuk eksperimen dengan cepat.

5. Pengembangan UI konsisten: Dengan menggunakan Flutter, Anda dapat menciptakan pengalaman pengguna yang konsisten di berbagai platform. Widget dan desain yang sama dapat digunakan di semua platform, sehingga memudahkan pengembangan dan mempertahankan konsistensi tampilan antarmuka pengguna.

1. Model Freemium: Perusahaan menyediakan versi dasar dari perangkat lunak secara gratis, tetapi menawarkan fitur-fitur premium atau tambahan yang lebih canggih dengan berlangganan berbayar. Pengguna dapat memilih untuk meningkatkan ke versi premium untuk mendapatkan manfaat lebih.

2. Layanan Berbayar: Meskipun perangkat lunak inti tersedia secara gratis, perusahaan dapat menghasilkan pendapatan dengan menawarkan layanan berbayar seperti dukungan pelanggan, pelatihan, konsultasi, atau integrasi khusus.

5. Pembayaran Donasi: Beberapa perusahaan memilih untuk mendapatkan dukungan finansial dari pengguna yang bersedia berdonasi. Ini dapat menjadi model yang efektif terutama jika pengguna merasa terhubung secara emosional dengan perangkat lunak tersebut.

7. Kemitraan dan Aliansi: Perusahaan pengembang perangkat lunak dapat menjalin kemitraan dengan perusahaan lain untuk menyediakan integrasi khusus atau solusi tambahan yang dapat dikenakan biaya.

8. Pemasaran Lintas: Perusahaan dapat menggunakan model perangkat lunak gratis sebagai alat pemasaran lintas untuk produk atau layanan lain yang mereka tawarkan. Perangkat lunak gratis dapat menarik perhatian calon pelanggan dan membantu memperkenalkan produk lain.

37. Contoh Metode Pengembangan di RPL(Rekayasa Perangkat Lunak) gmn si?


1. Linear Sequential Model (Model Sequential Linear)/ Model Waterfall

a. Model Waterfall

Menurut  (Pressman, Roger S. 2001) Metode Waterfall adalah suatu proses pengembangan perangkat lunak berurutan, di mana kemajuan dipandang sebagai terus mengalir ke bawah (seperti air terjun) melewati fase-fase perencanaan, pemodelan, implementasi (konstruksi), dan pengujian.

Tahapan metode Waterfall dapat digambarkan sebagai berikut: 

Dalam pengembangannya, metode Waterfall memiliki beberapa tahapan yang runtut: Requirement (analisis kebutuhan), Desain Sistem (system design), Coding & Testing, penerapan program dan pemeliharaan. Tahapan tahapan dari metode Waterfall adalah sebagai berikut :

Requirement Analysis

Pada tahap ini, pengembang sistem diperlukan suatu komunikasi yang bertujuan untuk memahami software yang diharapkan pengguna dan batasansoftware. Informasi ini biasanya dapat diperoleh melalui wawancara, survei atau diskusi. Informasi tersebut dianalisis untuk mendapatkan data yang dibutuhkan oleh pengguna.

2. System Design

Spesifikasi kebutuhan dari tahap pertama akan dipelajari dalam fase ini dan desain sistem disiapkan. Desain Sistem membantu dalam menentukan perangkat keras dan sistem persyaratan dan juga membantu dalam mendefinisikan arsitektur sistem secara keseluruhan.

3. Implementation

Pada tahap ini, sistempertama kali dikembangkan di program kecil yang disebut unit, yang terintegrasi dalam tahap berikutnya. Setiap unit dikembangkan dan diuji untuk fungsionalitas yang disebut sebagai Unit Testing.

4. Integration dan Testing

Semua unit yang dikembangkan dalam tahap implementasi diintegrasikan ke dalam sistem setelah pengujian masing-masing unit. Pasca integrasi seluruh sistem diuji untuk mengecek  setiap kesalahan dan kegagalan.

5. Operation dan Maintenance

Ini merupakan tahap terakhir dalam model waterfall. Softwareyang  sudah  jadi dijalankan  serta dilakukan pemeliharaan. Pemeliharaan termasuk dalam memperbaiki  kesalahan yang tidakditemukan pada langkah sebelumnya. Perbaikan implementasi unitsistem dan peningkatan jasa sistem sebagai kebutuhan baru



38. Apa itu pengembangan perangkat lunak agile, Dan bagaimana penerapannya.​


Pengembangan perangkat lunak agile adalah pendekatan dalam pengembangan perangkat lunak yang fleksibel, kolaboratif, dan berorientasi pada nilai pelanggan. Metode ini berfokus pada adaptasi terhadap perubahan yang terjadi selama proses pengembangan, memperhatikan interaksi dan komunikasi yang erat antara semua pemangku kepentingan terkait, serta mengutamakan pengerjaan perangkat lunak yang bermanfaat dan berkualitas.

Penerapan pengembangan perangkat lunak agile melibatkan beberapa prinsip dan praktik penting, antara lain sebagai berikut:

1. Kolaborasi Tim: Kolaborasi yang erat antara anggota tim pengembangan perangkat lunak maupun dengan pemangku kepentingan lainnya sangat penting. Tim bekerja bersama untuk merencanakan, mengerjakan, dan mengevaluasi proyek secara terus-menerus.

2. Deliverables Berkala: Pengembangan perangkat lunak dalam metode agile dilakukan dalam siklus iteratif atau sprint. Pada akhir setiap sprint, ada pengiriman deliverables yang siap digunakan atau diuji oleh pengguna atau pemangku kepentingan terkait.

3. Prioritas Tugas: Prioritas dalam pengembangan perangkat lunak agile ditentukan oleh pemangku kepentingan utama. Pekerjaan yang paling bernilai dan bermanfaat untuk pengguna atau pelanggan diberikan prioritas lebih tinggi.

4. Fleksibilitas dan Adaptabilitas: Metode ini dirancang untuk mengatasi perubahan yang mungkin terjadi selama proses pengembangan. Tim harus mampu merespons perubahan kebutuhan, pembaruan, atau perbaikan dengan cepat dan efektif.

5. Sifat Sederhana: Pengembangan perangkat lunak agile berfokus pada penyampaian fitur yang diperlukan dan berkualitas. Pendetilan yang berlebihan dan fitur yang tidak diperlukan dihindari, sehingga memastikan perangkat lunak tetap sederhana, mudah dipahami, dan dapat dioptimalkan.

6. Umpan Balik Kontinu: Umpan balik dari pengguna atau pemangku kepentingan lainnya sangat dihargai dan diintegrasikan ke dalam proses pengembangan secara teratur. Ini memastikan bahwa perangkat lunak terus diperbaiki dan disempurnakan sesuai kebutuhan pengguna yang sebenarnya.

7. Refleksi dan Pembelajaran: Tim terus belajar dari pengalaman sebelumnya dan melakukan refleksi terhadap hasil yang telah dicapai. Hal ini memungkinkan tim untuk terus meningkatkan praktik dan proses pengembangan perangkat lunak mereka.

Penerapan pengembangan perangkat lunak agile biasanya melibatkan penggunaan kerangka kerja atau metode seperti Scrum atau Kanban, di mana proyek dibagi menjadi sprint atau tugas dengan batasan waktu tertentu. Pemimpin tim (Scrum Master) membantu memfasilitasi proses dan memastikan tim bekerja secara efektif.

Dalam penerapannya, penting untuk memastikan kolaborasi yang baik antara tim dan pemangku kepentingan lainnya, menjaga komunikasi yang terbuka, dan menjaga fleksibilitas dalam merespons perubahan. Dengan mempraktikkan pendekatan ini, pengembangan perangkat lunak dapat lebih efisien, adaptif, dan menghasilkan produk yang lebih sesuai dengan kebutuhan pengguna.
